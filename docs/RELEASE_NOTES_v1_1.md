# EIT-DB v1.1.1 Release Notes

> 发布日期：2026-03-23  
> 类型：Patch（优化与 NoSQL 特色能力增强）

---

## 概述

v1.1.1 聚焦两个目标：

1. 优化跨适配器定时任务可用性，降低“某些后端不支持任务调度”带来的接入成本。
2. 提供更多可直接被应用层复用的 NoSQL 特色能力，减少重复实现与业务层胶水代码。

该版本不引入破坏性变更，保持 v1.1.0 兼容。

---

## 主要变更

### 1) 定时任务覆盖优化（跨适配器一致性）

- Repository 层新增统一回退调度策略：当 adapter 返回“不支持定时任务”时，可自动回退到内置调度器执行。
- 回退策略默认开启，并支持用户显式关闭。
- 目标是让任务能力在多数据库组合场景下具备稳定“可用下限”。

### 2) MongoDB 特色能力增强（面向应用层直用）

- 日志分析能力：
  - 热词分析
  - 规则分词（如 IP/URL/错误码/trace_id/hashtag）
  - 按级别、按时间窗口统计
- 文档工作流能力：
  - 草稿管理
  - 模板渲染
  - 模板预置库
  - 渲染安全策略（函数白名单、大小限制、缺失键处理）

这些能力可直接由应用层调用，减少重复实现日志解析与文档渲染管线。

### 3) Neo4j 社交/聊天模型增强（预置可复用模板）

- 社交预置模型：关注、好友、论坛等基础关系模型继续扩展。
- 聊天域模型：
  - 一对一聊天（可绑定关系门禁）
  - 群聊房间模型
  - 已读回执（READ_BY）
  - 管理状态（禁言/封禁）
- 消息全文检索：新增消息全文索引与检索模板。
- Emoji 关系模型更新：
  - 改为静态 Emoji 节点复用
  - 关系方向为 `(:Emoji)-[:INCLUDED_BY {index}]->(:ChatMessage)`
  - 通过 `index` 绑定消息中的 `{{index}}` 占位符

---

## 对应用层的价值

- 少写重复代码：日志分析、文档渲染、社交关系编排可直接调用 adapter 特色能力。
- 降低接入分叉：跨数据库任务调度有统一兜底，业务层不再需要按 adapter 大量分支处理。
- 更清晰的数据层边界：关系语义和访问规则前置到数据层模板，减少 API 层重复校验逻辑。
