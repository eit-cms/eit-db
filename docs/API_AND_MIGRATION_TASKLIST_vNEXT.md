# 新 API 与迁移任务清单（vNext）

目标：在协作层收尾后，推进后端无关 API、声明式配置与迁移体系升级。

更新时间：2026-04-25
状态：Ready after Collaboration Closeout Gate

## 1. 总体顺序

1. API 收敛先行：统一 Query/Exec 与事务语义抽象。
2. 配置收敛跟进：声明式优先与 TOML 清单落地。
3. 迁移治理收尾：兼容层、迁移工具、文档与门禁同步。

## 2. API 收敛任务

### A1. 统一执行入口

- [ ] 设计并冻结统一执行请求模型（读/写/参数/上下文）。
- [ ] 适配器层实现统一入口到 SQL/Cypher/AQL/聚合映射。
- [ ] 保留后端专属入口为兼容层并标注弃用计划。

验收标准：

1. 新增业务代码不需要按后端分支调用专属执行函数。
2. 执行结果结构在跨后端下具备统一字段语义。

### A2. 事务语义泛化

- [ ] 定义事务语义能力声明模型（多语句原子提交、隔离边界、限制）。
- [ ] 增加统一事务执行入口与能力检查。
- [ ] 在不支持事务语义的后端返回可诊断能力错误。

验收标准：

1. 事务开关由能力声明驱动，不由后端名称硬编码。
2. 事务失败信息包含可操作的限制说明与替代建议。

### A3. 关系语义驱动执行

- [ ] 将关系语义分级接入 Blueprint 路由约束。
- [ ] Query Module 声明关系能力要求并在编译前校验。
- [ ] 移除以 JOIN 覆盖率作为主决策依据的路径。

验收标准：

1. 路由决策日志可展示关系语义等级匹配过程。
2. 关系查询后端选择符合强/中/弱支持策略。

## 3. 声明式配置与 TOML 任务

### B1. RuntimeManifest 模型

- [ ] 定义 Manifest 顶层结构（adapters/collaboration/blueprint/schema/relations/query_modules）。
- [ ] 提供 TOML 解析与严格校验。
- [ ] 支持导出生效配置快照。

验收标准：

1. TOML 清单可独立驱动运行时初始化。
2. 校验失败可返回字段级错误定位。

### B2. 配置兼容桥接

- [ ] 提供 Manifest 到现有 Config 的桥接层。
- [ ] 支持环境变量局部覆盖并记录来源。
- [ ] 保持 YAML/JSON 与构建式入口在兼容期可用。

验收标准：

1. 同一拓扑在 TOML 与旧配置路径下行为一致。
2. 最终生效配置可追踪每个字段来源。

### B3. 文档与脚手架

- [ ] CLI 或模板默认输出 TOML 配置骨架。
- [ ] 更新快速上手文档到声明式优先路径。
- [ ] 在发布说明中明确兼容层生命周期。

## 4. 迁移执行任务

### C1. 调用点盘点

- [ ] 统计后端专属入口调用点并按模块归类。
- [ ] 建立迁移看板（模块、负责人、风险、状态）。
- [ ] 冻结新增后端专属调用。

### C2. 增量迁移

- [ ] 第一批迁移核心查询路径到统一入口。
- [ ] 第二批迁移事务边界到统一事务入口。
- [ ] 第三批迁移关系语义查询到声明式关系编译路径。

### C3. 兼容层退场

- [ ] 为兼容 API 增加迁移告警与可关闭开关。
- [ ] 在目标版本标记 deprecated，下一阶段计划移除。
- [ ] 提供自动化检查规则阻止新增兼容层调用。

## 5. 测试与质量门禁

### D1. 测试矩阵

- [ ] 统一执行入口跨后端一致性测试。
- [ ] 事务语义能力测试（支持与不支持场景）。
- [ ] TOML 配置解析与桥接一致性测试。
- [ ] 迁移回归测试（旧 API 与新 API 并存期）。

### D2. 发布门禁

- [ ] CI 增加后端无关 API 兼容回归任务。
- [ ] CI 增加 TOML 配置校验任务。
- [ ] 发布前生成迁移影响报告与风险清单。

## 6. 里程碑与 Gate

### Gate-API-1

进入 TOML 配置主线前需要满足：

1. 统一执行入口可覆盖主查询路径。
2. 事务语义模型与错误模型冻结。

### Gate-API-2

进入兼容层退场前需要满足：

1. 主业务模块迁移率不低于 80%。
2. 迁移期回归稳定运行 1 个迭代周期。

### Gate-API-3

进入下一版本冻结前需要满足：

1. 文档、迁移指南、发布说明一致。
2. 兼容层退场计划与时间窗对外明确。
